# Contributing

- Search existing issues and pull requests before filing a new one.
- Follow the contribution steps: fork, branch, implement, and open a PR.
- Provide a clear description and rationale for changes.
- Consider supporting the project if it is helpful to you.

## Examples

### Good

Search existing issues, then open a focused PR with a clear description and rationale.

### Bad

Open a PR without context or skip checking existing issues first.
